3 · the honesty rule, in the source

VERBATIM
MakeMyTeam/backend/app/pipeline/seam.py
HONESTY FRAMING (BUILD_GUIDE §5 / Golden Rule #6 — non-negotiable): a seam is
EXERCISED in the sandbox — agent A really runs and produces an artifact, agent
B really consumes THAT artifact and produces a deliverable; that chain is the
real integration signal. But the SCORE is an LLM judge over acceptance
criteria: it is rubric-scored, NOT execution-verified. Every seam result this
module emits therefore carries execution_verified: False and the SEAM_NOTE
badge, and NO code path here ever sets execution_verified True. Seam scores
must never be presented as execution-verified or blended into the Level-1
executed number (team_run keeps them in a separate `level2` section).
